## 一、设计目标

### 1.1 核心理念

**给"如果当初"祛魅**

人最擅长美化自己没选的那条路。镜中调用真实世界的数据，告诉用户那条路上的真实状况，不是为了证明"你选对了"或"你选错了"，而是帮用户把被浪漫化的路还原成它本来的样子。

### 1.2 设计原则

```
┌─────────────────────────────────────────────────────────────────┐
│                    真实世界数据设计原则                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 用事实对抗想象                                              │
│     · 用户想象中的路往往是美化的                                │
│     · 真实数据可以帮助还原那条路的真实状况                       │
│     · 不是为了证明对错，而是让用户看到更多信息                   │
│                                                                 │
│  2. 只用公开可获取的数据                                        │
│     · 天气、空气质量、房价、政策等公开数据                      │
│     · 不爬取私人社交媒体                                        │
│     · 不使用非公开的职场社交数据                                │
│                                                                 │
│  3. 按需查询，不预爬取                                          │
│     · 用户触发推演时才查询                                      │
│     · 查询结果缓存，减少重复请求                                │
│     · 保护用户隐私，不泄露推演内容                              │
│                                                                 │
│  4. 数据融入叙事，不堆砌                                        │
│     · 数据不是报告，是对话的一部分                              │
│     · 用自然语言呈现数据                                        │
│     · 数据服务于叙事，不是目的                                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1.3 数据接入范围

```
┌─────────────────────────────────────────────────────────────────┐
│                    数据接入范围矩阵                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  【第一优先级 - 核心场景必需】                                   │
│  ─────────────────────────────────────────────────────────────  │
│  · 天气历史数据：还原"当时如果是那个城市"                        │
│  · 空气质量历史：还原环境状况                                    │
│  · 房价/租金历史：还原经济决策背景                               │
│  · 城市生活成本：还原生活状况                                    │
│                                                                 │
│  【第二优先级 - 重要补充】                                       │
│  ─────────────────────────────────────────────────────────────  │
│  · 公司动态（公开信息）：还原职业选择背景                        │
│  · 行业趋势数据：还原行业状况                                    │
│  · 重大新闻事件：还原时代背景                                    │
│  · 政策变化：还原政策环境                                        │
│                                                                 │
│  【第三优先级 - 体验增强】                                       │
│  ─────────────────────────────────────────────────────────────  │
│  · 汇率历史：涉及海外决策时                                      │
│  · 航班/票价历史：涉及旅行决策时                                 │
│  · 就业市场数据：涉及求职决策时                                  │
│                                                                 │
│  【不接入】                                                      │
│  ─────────────────────────────────────────────────────────────  │
│  · 私人社交媒体数据                                              │
│  · 非公开的职场社交数据                                          │
│  · 需要用户授权的第三方私人数据                                  │
│  · 任何可能侵犯隐私的数据                                        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 二、数据源设计

### 2.1 天气与空气质量

```
┌─────────────────────────────────────────────────────────────────┐
│                    天气与空气质量数据                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  【数据源】                                                      │
│  ─────────────────────────────────────────────────────────────  │
│  · 国内：和风天气API、中国气象局公开数据                         │
│  · 国外：OpenWeatherMap、Visual Crossing                         │
│  · 空气质量：AQICN、World Air Quality Index                      │
│                                                                 │
│  【应用场景】                                                    │
│  ─────────────────────────────────────────────────────────────  │
│  · 推演"如果当初去了那个城市"                                   │
│  · 还原当时的天气状况                                           │
│  · 对比用户实际所在地的天气                                      │
│                                                                 │
│  【查询时机】                                                    │
│  ─────────────────────────────────────────────────────────────  │
│  · 平行人生推演涉及地理位置变化时                               │
│  · 用户询问"那年那时的天气"时                                   │
│                                                                 │
│  【数据呈现示例】                                                │
│  ─────────────────────────────────────────────────────────────  │
│                                                                 │
│  推演："如果当初去了深圳"                                       │
│                                                                 │
│  系统查询：2024年深圳天气数据                                    │
│  ─────────────────────────────────────────────────────────────  │
│  · 2024年深圳平均气温：23.5°C                                   │
│  · 2024年深圳高温天数（>35°C）：47天                             │
│  · 2024年深圳空气质量优良天数：312天                             │
│  （对比）2024年北京空气质量优良天数：256天                       │
│                                                                 │
│  融入叙事：                                                      │
│  "2024年深圳的夏天很热，有47天气温超过35度。                    │
│   空气确实比北京好，蓝天比较多。                                 │
│   但你当时最在意的不就是天气吗？这一点倒是对的。"               │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 三、查询与缓存策略

### 3.1 查询时机

```
┌─────────────────────────────────────────────────────────────────┐
│                      数据查询时机                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  【触发查询的场景】                                              │
│  ─────────────────────────────────────────────────────────────  │
│                                                                 │
│  场景一：平行人生推演                                            │
│  用户触发推演 → 提取时间点和地点 → 并行查询相关数据              │
│                                                                 │
│  场景二：决策辅助分析                                            │
│  用户面临决策 → 查询相关城市/行业/公司数据                       │
│                                                                 │
│  场景三：用户主动询问                                            │
│  用户："那年那时候的天气怎么样？" → 直接查询                     │
│                                                                 │
│  【不触发查询】                                                  │
│  ─────────────────────────────────────────────────────────────  │
│  · 普通对话，不涉及具体时间地点                                  │
│  · 用户未触发推演或决策分析                                      │
│  · 避免无意义的预爬取                                            │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 查询流程

```
用户触发推演/分析
        │
        ▼
┌─────────────────────────────────────────────────────────────────┐
│ Step 1: 提取查询参数                                             │
│ ─────────────────────────────────────────────────────────────── │
│                                                                 │
│  从推演请求中提取：                                              │
│  · 时间点（用户提到的时间）                                      │
│  · 地点（城市/国家）                                             │
│  · 相关实体（公司名、行业名）                                    │
│  · 查询类型（天气/房价/公司动态/政策等）                         │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
        │
        ▼
┌─────────────────────────────────────────────────────────────────┐
│ Step 2: 检查缓存                                                 │
│ ─────────────────────────────────────────────────────────────── │
│                                                                 │
│  查询缓存：                                                      │
│  · 历史天气数据 → 永久缓存（历史数据不变）                       │
│  · 历史空气质量 → 永久缓存                                       │
│  · 公司动态 → 缓存30天，到期检查更新                            │
│  · 房价走势 → 缓存7天                                            │
│  · 政策变化 → 缓存90天                                           │
│                                                                 │
│  命中缓存 → 直接使用                                             │
│  未命中 → 调用API查询                                            │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
        │
        ▼
┌─────────────────────────────────────────────────────────────────┐
│ Step 3: 并行查询                                                 │
│ ─────────────────────────────────────────────────────────────── │
│                                                                 │
│  同时发起多个查询请求：                                          │
│  · 天气API                                                      │
│  · 房价API                                                      │
│  · 新闻搜索API                                                  │
│  · 其他相关API                                                  │
│                                                                 │
│  设置超时：每个查询最多5秒                                       │
│  部分失败不影响整体推演                                          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
        │
        ▼
┌─────────────────────────────────────────────────────────────────┐
│ Step 4: 数据整合                                                 │
│ ─────────────────────────────────────────────────────────────── │
│                                                                 │
│  将查询结果整合为推演上下文：                                    │
│  · 提取关键数据点                                                │
│  · 对比用户实际情况                                              │
│  · 准备融入叙事的数据片段                                        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
        │
        ▼
┌─────────────────────────────────────────────────────────────────┐
│ Step 5: 缓存结果                                                 │
│ ─────────────────────────────────────────────────────────────── │
│                                                                 │
│  将查询结果存入缓存，供后续使用                                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3.3 缓存策略

```
┌─────────────────────────────────────────────────────────────────┐
│                      缓存策略详情                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  【永久缓存】                                                    │
│  ─────────────────────────────────────────────────────────────  │
│  数据类型：历史天气、历史空气质量、历史事件                      │
│  理由：这些数据不会变化                                          │
│  存储位置：本地 + 云端                                           │
│  存储格式：年份_城市_数据类型                                    │
│                                                                 │
│  【短期缓存】                                                    │
│  ─────────────────────────────────────────────────────────────  │
│  数据类型：公司动态、行业趋势                                    │
│  缓存时间：30天                                                  │
│  更新策略：到期后重新查询                                        │
│                                                                 │
│  【实时缓存】                                                    │
│  ─────────────────────────────────────────────────────────────  │
│  数据类型：当前房价、当前生活成本                                │
│  缓存时间：7天                                                   │
│  更新策略：每次查询检查缓存时间                                  │
│                                                                 │
│  【缓存键设计】                                                  │
│  ─────────────────────────────────────────────────────────────  │
│  格式：data_type:location:year_month                             │
│  示例：                                                          │
│  · weather:shenzhen:2024_06                                     │
│  · housing:hangzhou:2021                                        │
│  · company:didi:2023                                            │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 四、数据融入叙事

### 4.1 叙事融合原则

```
┌─────────────────────────────────────────────────────────────────┐
│                    数据融入叙事原则                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  【不是报告，是对话】                                            │
│  ─────────────────────────────────────────────────────────────  │
│  ❌ 错误：                                                       │
│  "根据数据显示，2024年深圳平均气温23.5°C，高温天数47天，         │
│   空气质量优良天数312天，比北京多56天……"                         │
│                                                                 │
│  ✅ 正确：                                                       │
│  "深圳的夏天确实热，那你有47天要顶着35度以上的高温。             │
│   不过蓝天是比北京多。"                                          │
│                                                                 │
│  ─────────────────────────────────────────────────────────────  │
│                                                                 │
│  【数据服务于叙事，不是目的】                                    │
│  ─────────────────────────────────────────────────────────────  │
│  数据不是用来证明用户的对错，而是补充信息                        │
│  ─────────────────────────────────────────────────────────────  │
│                                                                 │
│  ❌ 错误：                                                       │
│  "房价跌了15%，所以你没买是对的。"                               │
│                                                                 │
│  ✅ 正确：                                                       │
│  "杭州房价后来跌了15%左右。                                     │
│   这是事实，但它不能告诉你现在该不该买——                        │
│   因为未来还会怎么走，没人知道。"                                │
│                                                                 │
│  ─────────────────────────────────────────────────────────────  │
│                                                                 │
│  【简洁，不过度堆砌】                                            │
│  ─────────────────────────────────────────────────────────────  │
│  每个推演节点最多融入2-3个数据点                                 │
│  选择最相关的数据                                                │
│  ─────────────────────────────────────────────────────────────  │
│                                                                 │
│  ❌ 错误：                                                       │
│  "2024年深圳GDP增长5.8%，人口增加12万，                          │
│   平均工资12500元，地铁新增2条线，                               │
│   房价均价7.2万，写字楼空置率25%……"                              │
│                                                                 │
│  ✅ 正确：                                                       │
│  "深圳的机会确实比以前多，工资也还行。                           │
│   但房价7万多一平，对刚去的人来说压力不小。"                      │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4.2 数据来源标注

```
┌─────────────────────────────────────────────────────────────────┐
│                    数据来源标注规范                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  【轻度标注】                                                    │
│  ─────────────────────────────────────────────────────────────  │
│  在叙事中轻带数据来源，不破坏对话流畅性                          │
│                                                                 │
│  示例：                                                          │
│  "那家公司2024年做了B轮，估值大概翻了倍。                         │
│   这是公开新闻能查到的。"                                        │
│                                                                 │
│  "深圳2024年有47天超过35度，                                     │
│   比北京热多了。"                                                │
│  （天气数据不需要特别标注，因为可信度高）                         │
│                                                                 │
│  【不确定时说明】                                                │
│  ─────────────────────────────────────────────────────────────  │
│  数据不完整或有争议时，向用户说明                                │
│                                                                 │
│  示例：                                                          │
│  "那家公司的具体情况我查不到太多公开信息，                        │
│   只知道2023年融了A轮。                                          │
│   后来怎么样就不清楚了。"                                        │
│                                                                 │
│  "房价数据只能查到均价，具体小区可能差距挺大的。"                 │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 五、隐私与合规

### 5.1 数据边界

```
┌─────────────────────────────────────────────────────────────────┐
│                      数据使用边界                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  【使用原则】                                                    │
│  ─────────────────────────────────────────────────────────────  │
│  · 只使用公开可获取的数据                                        │
│  · 不爬取任何私人数据                                            │
│  · 不分析用户的社交关系                                          │
│  · 不追踪用户的网络行为                                          │
│                                                                 │
│  【具体边界】                                                    │
│  ─────────────────────────────────────────────────────────────  │
│  · 天气、空气质量：公开数据，可使用                              │
│  · 房价指数：公开数据，可使用                                    │
│  · 公司动态：仅公开新闻报道和公告                                │
│  · 行业报告：公开报告，可使用                                    │
│  · 职场社交数据（脉脉、领英）：不使用                            │
│  · 朋友圈、微博等私人社交：不使用                                │
│  · 任何需要登录才能查看的数据：不使用                            │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 5.2 查询隐私

```
┌─────────────────────────────────────────────────────────────────┐
│                      查询隐私保护                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  【查询过程】                                                    │
│  ─────────────────────────────────────────────────────────────  │
│  · 用户发起推演 → 系统后台查询数据                               │
│  · 查询请求不包含用户身份信息                                    │
│  · 第三方API只能看到"有人查询了XX城市XX年的天气"                 │
│  · 第三方无法知道是谁在查询，也无法知道查询目的                  │
│                                                                 │
│  【数据存储】                                                    │
│  ─────────────────────────────────────────────────────────────  │
│  · 查询结果缓存在云端                                            │
│  · 缓存数据不包含用户身份                                        │
│  · 只缓存数据本身，不缓存查询关联的用户信息                      │
│                                                                 │
│  【用户可控】                                                    │
│  ─────────────────────────────────────────────────────────────  │
│  · 用户可以在设置中关阂数据查询功能                              │
│  · 关闭后，推演将不包含真实世界数据                              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 六、降级与容错

### 6.1 查询失败处理

```
┌─────────────────────────────────────────────────────────────────┐
│                    查询失败处理                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  【部分失败】                                                    │
│  ─────────────────────────────────────────────────────────────  │
│  如果部分数据查询失败，不影响整体推演                            │
│                                                                 │
│  示例：                                                          │
│  天气数据成功，房价数据失败                                      │
│  → 推演正常进行，只是少了一部分数据支撑                          │
│  → 系统自然过渡："天气方面……房价的具体数据我没查到……"           │
│                                                                 │
│  【全部失败】                                                    │
│  ─────────────────────────────────────────────────────────────  │
│  如果所有数据查询都失败，推演仍然可以继续                        │
│                                                                 │
│  示例：                                                          │
│  "外部数据暂时查不到，我先根据你的情况推演一下——                 │
│   如果你去了那个城市，可能会遇到……"                              │
│                                                                 │
│  【数据缺乏】                                                    │
│  ─────────────────────────────────────────────────────────────  │
│  某些数据可能根本不存在                                          │
│                                                                 │
│  示例：                                                          │
│  "那家小公司没有太多公开数据，具体情况我也不清楚。                │
│   只能根据一般创业公司的情况来推演了。"                           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 6.2 数据不确定处理

```
┌─────────────────────────────────────────────────────────────────┐
│                    数据不确定处理                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  【数据范围】                                                    │
│  ─────────────────────────────────────────────────────────────  │
│  提供数据范围而非精确数字，降低误导风险                          │
│                                                                 │
│  示例：                                                          │
│  ❌ "2024年深圳房价7.2万/平"                                     │
│  ✅ "2024年深圳房价大概在7万左右"                                │
│                                                                 │
│  【不确定性说明】                                                │
│  ─────────────────────────────────────────────────────────────  │
│  数据来源不明确或有争议时，向用户说明                            │
│                                                                 │
│  示例：                                                          │
│  "网上说法不太一样，有说涨了，有说跌了。                          │
│   总之那个片区的房价波动挺大的。"                                │
│                                                                 │
│  【避免过度解读】                                                │
│  ─────────────────────────────────────────────────────────────  │
│  不替用户下结论，只提供信息                                      │
│                                                                 │
│  示例：                                                          │
│  ❌ "房价跌了，所以你没买是对的"                                 │
│  ✅ "房价跌了大概15%。这是事实，但它意味着什么，得你自己判断。"   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```
