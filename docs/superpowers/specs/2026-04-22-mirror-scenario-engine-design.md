## 一、引擎概述

### 1.1 核心定位

情景推演引擎是镜中的"未来望远镜"和"平行时空模拟器"：

- **平行人生推演**：如果当初选了另一条路，那条路上会有什么
- **决策辅助分析**：客观分析当下的决策，呈现多种可能性

### 1.2 设计原则

```
┌─────────────────────────────────────────────────────────────────┐
│                      推演引擎核心原则                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 不预测未来，只呈现可能性                                     │
│     · 输出是"可能这样"，不是"将会这样"                          │
│     · 明确标注不确定性来源                                       │
│                                                                 │
│  2. 不替用户决策，只确保信息充分                                 │
│     · 提供分析，不提供结论                                       │
│     · 最终选择权永远在用户                                       │
│                                                                 │
│  3. 基于画像，关联用户真实情况                                   │
│     · 推演必须绑定用户的价值观、风险承受、能力边界               │
│     · 不做脱离用户处境的泛化推演                                 │
│                                                                 │
│  4. 客观分析，不做情感施压                                       │
│     · 展示事实和可能性，不带压迫感                               │
│     · 帮助用户理解，而非挑战用户                                 │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 二、平行人生推演

### 2.1 核心概念

**什么是"平行人生推演"？**

用户提出一个"如果当初"的问题，系统基于用户画像、记忆库和真实世界数据，推演出那条未走的路上可能发生的事。

**核心特点：**

- 绑定用户真实情况，不做泛化推演
- 嵌入真实世界数据，给"如果当初"祛魅
- 以叙事方式呈现，用户可互动分支
- 生成可视化世界线树，记录推演历史

### 2.2 节点结构设计

#### 2.2.1 节点类型

```
┌─────────────────────────────────────────────────────────────────┐
│                        节点类型定义                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  【起源节点 Origin Node】                                        │
│  · 定义：用户现实人生中选择分支的起点                            │
│  · 内容：真实发生的选择事件 + 当时情境                           │
│  · 示例：2024年3月，你选择了辞职                                 │
│  · 特点：基于记忆库真实数据                                      │
│                                                                 │
│  【分支节点 Branch Node】                                        │
│  · 定义：推演中的关键选择点                                      │
│  · 内容：推演情境中的选择 + 后果预览                             │
│  · 示例：入职第三个月，公司调整业务方向，你选择留下/离开          │
│  · 特点：由规则框架生成，用户可选择                              │
│                                                                 │
│  【结局节点 Ending Node】                                        │
│  · 定义：推演的时间线终点或暂停点                                │
│  · 内容：该推演路径的阶段性结局                                  │
│  · 示例：一年后，你成为小团队负责人，但公司仍在融资困境中         │
│  · 特点：可继续推演或结束                                        │
│                                                                 │
│  【平行节点 Parallel Node】                                      │
│  · 定义：从同一分支点延伸出的不同选择路径                        │
│  · 内容：同一选择点的另一条路径                                  │
│  · 示例：同一起点，选择留下而不是离开的推演                       │
│  · 特点：与当前节点形成平行对照                                  │
│                                                                 │
│  【决策中节点 Pending Node】                                      │
│  · 定义：用户正在考虑但未做出的未来决定                           │
│  · 内容：决策情境 + 待选项 + 分析维度                             │
│  · 示例：正在考虑是否接受新工作机会                               │
│  · 特点：节点边框为灰色实线+问号，决策后转为实际节点              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

#### 2.2.2 推演节点关系类型

**当用户发起推演时，需确认新推演节点与已有节点的关系：**

```
┌─────────────────────────────────────────────────────────────────┐
│                      推演节点关系类型                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  【子节点分叉 Child Branch】                                     │
│  ─────────────────────────────────────────────────────────────  │
│  定义：从已有节点延伸出的推演分支                                │
│  触发场景："如果当初在这个决定之后，选择了另一条路"                │
│  示例：                                                          │
│  已有节点：2023年5月 辞职                                        │
│  推演：辞职后去了另一家公司                                      │
│  关系：新推演节点作为"辞职"节点的子节点                          │
│  世界线显示：从已有节点延伸出的分支线                            │
│                                                                 │
│         ─────────●────────────────                               │
│                    │                                             │
│                    ├──────◇ 推演：去了另一家公司                  │
│                    │                                             │
│         [2023.5 辞职]                                           │
│                                                                 │
│  ─────────────────────────────────────────────────────────────  │
│                                                                 │
│  【前序分叉 Pre-branch】                                         │
│  ─────────────────────────────────────────────────────────────  │
│  定义：在已有节点之前的分叉，推演"如果这个决定没有发生"          │
│  触发场景："如果当初没有做这个决定"                              │
│  示例：                                                          │
│  已有节点：2023年5月 辞职                                        │
│  推演：如果当初没有辞职                                          │
│  关系：新推演节点定位在已有节点之前，形成平行分支                │
│  世界线显示：与已有节点平行的虚线分支                            │
│                                                                 │
│                    ┌──────◇ 推演：没有辞职                       │
│                    │                                             │
│         ─────────●┼───────────────                               │
│                    │                                             │
│         [2023.5 辞职]                                           │
│                                                                 │
│  ─────────────────────────────────────────────────────────────  │
│                                                                 │
│  【并行节点 Parallel Node】                                       │
│  ─────────────────────────────────────────────────────────────  │
│  定义：与已有节点无关的独立推演                                  │
│  触发场景：同一时期的另一件完全不同的事                          │
│  示例：                                                          │
│  已有节点：2023年5月 辞职                                        │
│  推演：如果当初去旅行了                                          │
│  关系：新推演节点独立，与已有节点无直接关联                      │
│  世界线显示：同一时间段的不同分支                                │
│                                                                 │
│         ─────────●───────────────                               │
│                    │                                             │
│         [2023.5 辞职]                                           │
│                                                                 │
│         ─────────◇───────────────                               │
│                    │                                             │
│         [推演：去旅行了]                                         │
│                                                                 │
│  ─────────────────────────────────────────────────────────────  │
│                                                                 │
│  【未来决策节点 Future Decision】                                 │
│  ─────────────────────────────────────────────────────────────  │
│  定义：用户正在考虑的尚未发生的决定                              │
│  触发场景："我正在考虑要不要..."                                 │
│  示例：正在考虑是否接受新工作机会                                │
│  关系：位于世界线末尾                                            │
│  世界线显示：灰色实线边框+问号标记                               │
│  决策后：转为实际结果节点（决定类）                              │
│                                                                 │
│         ─────────●───────────────◎                              │
│                    │                    │                        │
│               [历史节点]          [决策中...]                    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**节点关系判断流程：**

```
用户发起推演请求
        │
        ▼
系统解析推演时间点和内容
        │
        ▼
检查该时间点是否已有节点
        │
        ├── 无节点 → 直接创建新推演节点
        │
        └── 有节点 → 进入关系确认流程
                    │
                    ▼
            展示已有节点信息
                    │
                    ▼
            用户选择关系类型：
            ├── 子节点分叉（从这个节点延伸）
            ├── 前序分叉（在这个节点之前分叉）
            └── 并行节点（独立新推演）
                    │
                    ▼
            根据选择创建节点关系
                    │
                    ▼
            进入推演对话
```

### 2.4 节点叙事风格规范

#### 2.4.1 叙事原则

```
┌─────────────────────────────────────────────────────────────────┐
│                        叙事风格规范                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ✅ 正确示例                                                     │
│                                                                 │
│  "入职三个月了，公司正在进行第二轮面试。                         │
│   你的入职让产品进度加快了不少，老板私下跟你说，                 │
│   如果这轮融资成功，你会有0.5%的期权。                           │
│   2024年6月，赛道整体热度上升，你的公司在同类项目中排名前三。"    │
│                                                                 │
│  ❌ 错误示例                                                     │
│                                                                 │
│  "后来你才知道，入职三个月的那段日子，会成为你人生的转折点……     │
│   那个炎热的夏天，你做出了改变命运的选择……"                      │
│   （太文学化，不直接）                                           │
│                                                                 │
│  ─────────────────────────────────────────────────────────────── │
│                                                                 │
│  核心规则：                                                      │
│  1. 直接陈述事实，不用"后来才知道"等后见之明                     │
│  2. 短句为主，保持对话体                                         │
│  3. 数据放在叙事中，不加过多渲染                                 │
│  4. 绑定用户画像：提到用户的具体情况（房贷、家庭、压力点）        │
│  5. 嵌入真实世界数据时，标注时间点                               │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

#### 2.4.2 叙事模板

```

示例：

 公司正在准备A轮融资，你负责的模块已经上线两个月。"

【选择呈现模板】

事件：{触发选择的事件}
选项A：{选项描述} → {简要后果预览}
选项B：{选项描述} → {简要后果预览}

示例：
```

## 三、动态漫画生成

### 3.1 设计理念

**为什么是动态漫画，而不是视频？**

| 维度 | 动态漫画 | AI视频 |
|-----|---------|-------|
| 生成稳定性 | 高，分镜可控 | 低，易出现不一致 |
| 生成速度 | 快（秒级） | 慢（分钟级） |
| 视觉风格一致性 | 高，风格固定 | 低，风格漂移 |
| 用户信息内嵌 | 容易（文字、符号） | 困难（需要训练） |
| 成本 | 低 | 高 |
| 交互性 | 高，可暂停选择 | 低，需播完 |

### 6.2 规则框架详解

规则框架采用混合架构：规则模板 + LLM生成

```
┌─────────────────────────────────────────────────────────────────┐
│                        规则框架示例                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  【分支规划规则】                                                │
│                                                                 │
│  规则1：职业选择类推演                                           │
│  IF 推演类型 == "职业选择" THEN                                 │
│     分支节点 = [入职适应期, 3个月考核, 6个月选择点, 1年结局]   │
│     每个节点选择数 = 2-3                                        │
│     必须嵌入外部数据 = 行业趋势, 公司动态                       │
│                                                                 │
│  规则2：城市选择类推演                                           │
│  IF 推演类型 == "城市选择" THEN                                 │
│     分支节点 = [搬家适应期, 工作寻找, 社交建立, 半年结局]       │
│     每个节点选择数 = 2                                          │
│     必须嵌入外部数据 = 生活成本, 就业市场, 气候环境             │
│                                                                 │
│  ─────────────────────────────────────────────────────────────── │
│                                                                 │
│  【概率计算规则】                                                │
│                                                                 │
│  规则3：概率范围                                                 │
│     积极/中性/挑战情景 概率和 = 100%                            │
│     积极 < 50%（避免过于乐观）                                   │
│     挑战 > 15%（必须呈现风险）                                   │
│                                                                 │
│  规则4：概率影响因素                                             │
│     基础概率（行业平均）                                         │
│     + 用户画像修正（风险承受高 → 积极概率+5%）                  │
│     + 外部数据修正（行业下滑 → 挑战概率+10%）                    │
│                                                                 │
│  ─────────────────────────────────────────────────────────────── │
│                                                                 │
│  【画像关联规则】                                                │
│                                                                 │
│  规则5：风险承受底线                                             │
│     IF 用户风险承受 < 当前选择风险 THEN                         │
│        生成警告提示                                             │
│        提供"如何降低风险"的分支持                               │
│        END IF                                                   │
│                                                                 │
│  规则6：价值观冲突检测                                           │
│     IF 当前选择 与 用户价值观.家庭优先 冲突 THEN                │
│        在推演中强调家庭因素影响                                  │
│        生成"与家人沟通"的交互点                                  │
│        END IF                                                   │
│                                                                 │
│  ─────────────────────────────────────────────────────────────── │
│                                                                 │
│  【外部数据嵌入规则】                                            │
│                                                                 │
│  规则7：数据嵌入时机                                             │
│     每个推演节点 至少嵌入1条真实数据                            │
│     嵌入位置 = 叙事中段，轻轻带过                                │
│     数据来源标注 = 简括号注释                                    │
│                                                                 │
│  规则8：数据嵌入格式                                             │
│     "2024年6月，赛道整体热度上升 [行业报告]"                    │
│     而非：                                                       │
│     "根据XX行业报告显示，2024年6月赛道热度上升了XX%"            │
│     （过于书面）                                                 │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```
