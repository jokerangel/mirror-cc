# 镜中（Mirror）- 产品设计总览

> 文档版本：v1.0
> 最后更新：2026-04-26
> 状态：设计完成

---

## 产品定位

**镜中** — 一面照向人生另一种可能的镜子。

通过长期对话建立对用户的深度理解，帮助用户看见那个"看不见的自己"。

---

## 核心理念

- **不预测未来，只呈现可能性**
- **不替用户决策，只确保信息充分**
- **不评判选择，只帮看见选择背后的自己**
- **数据只属于用户，加密存储，随时可销毁**

### 第一阶段：长期对话系统

**文档**：[2026-04-19-mirror-longterm-dialogue-design.md](2026-04-19-mirror-longterm-dialogue-design.md)

**核心模块**：

| 模块 | 说明 |
|------|------|
| 双核架构 | 记忆库 + 用户画像，双向校验 |
| 画像结构 | 12维度用户画像 + 5层Persona + 镜中专有层 |
| 记忆库 | 四层结构：原始层、关键片段层、时间线层、索引层 |

| 苏格拉底对话 | 5-10次完成初始画像，单线深入式 |
| 数据安全 | 本地计算 + 云端加密，服务端无法读取 |

**关键设计决策**：

| 决策 | 选择 |
|------|------|
| 数据架构 | 双核驱动（记忆库+画像层） |
| 存储方案 | 本地计算+云端加密 |
| 关系图谱粒度 | 深度记录（含情绪上下文） |
| 删除机制 | 进入隐藏层，不物理删除 |

### 第二阶段：情景推演引擎

**文档**：[2026-04-22-mirror-scenario-engine-design.md](2026-04-22-mirror-scenario-engine-design.md)

**核心模块**：

| 模块 | 说明 |
|------|------|
| 平行人生推演 | "如果当初选了另一条路会怎样" |
| 决策辅助分析 | 客观推演+利弊分析+行动建议 |
| 节点结构 | 起源节点、分支节点、结局节点、平行节点 |
| 叙事规范 | 直接陈述事实，不文学化包装 |
| 动态漫画 | 4种视觉风格，4幕叙事结构 |
| 世界线树 | 可视化所有推演历史，支持返回任意节点 |

**四种视觉风格**：

| 风格 | 适用场景 |
|------|---------|
| 怀旧胶片 | 回忆、温情、家庭相关 |
| 冷峻纪实 | 职业选择、理性分析 |
| 碎片记忆 | 情感波动、内心冲突 |
| 水墨写意 | 人生重大选择、哲学思考 |

### 第三阶段：功能模块

**文档**：[2026-04-22-mirror-features-design.md](2026-04-22-mirror-features-design.md)

**核心模块**：

| 模块 | 说明 |
|------|------|
| 用户画像（冷启动） | 快速生成画像产生"哇"时刻，带钩子指向下一步 |
| 关键事项记录 | 轻便记录人生关键时刻，存档动效仪式感 |
| 遗憾管理 | 识别、追踪、化解遗憾，推动行动或帮助释怀 |

**用户分层**：

| 类型 | 特点 | 关键任务 |
|------|------|---------|
| 探索型（大部分） | "试试看能干嘛" | 快速产生"哇"时刻 |
| 需求型（少部分） | "我有问题需要帮助" | 体现实际价值，留存 |

**关键事项记录流程**：

```
对话中识别 → 重要性判断 → 深入沟通 → 存档动效 → 世界线节点
```

**节点操作约束**：
- 可补充细节（生成反馈节点关联）
- 不可修改核心事实
- 删除走隐藏层

### 第四阶段：真实世界数据接入

**文档**：[2026-04-23-mirror-realworld-data-design.md](2026-04-23-mirror-realworld-data-design.md)

**核心模块**：

| 模块 | 说明 |
|------|------|
| 天气与空气质量 | 还原"如果去了那个城市"的天气状况 |
| 房价与租金 | 还原购房/租房决策背景 |
| 公司与行业动态 | 还原职业选择背景 |
| 政策与重大事件 | 还原时代背景 |

**设计原则**：

| 原则 | 说明 |
|------|------|
| 按需查询 | 用户触发推演时才查询，不预爬取 |
| 公开数据 | 只用公开可获取的数据，不用私人数据 |
| 叙事融合 | 数据融入对话，不做报告式堆砌 |
| 隐私保护 | 查询不包含用户身份，第三方无法知道查询目的 |

**数据融入叙事示例**：

❌ 报告化：
"根据数据显示，2024年深圳平均气温23.5°C，高温天数47天……"

✅ 对话化：
"深圳的夏天确实热，那你有47天要顶着35度以上的高温。不过蓝天是比北京多。"

## 核心交互流程

### 新用户首次使用

```
打开App
    │
    ▼
导入AI聊天记录 / 快问快答
    │
    ▼
生成初始画像（"哇时刻"）
    │
    ▼
画像带钩子指向话题
    │
    ▼
用户选择感兴趣的话题
    │
    ▼
开始深入对话 / 触发推演
    │
    ▼
存档动效 → 世界线生长
    │
    ▼
持续对话 + 记录关键事项
    │
    ▼
画像持续完善 + 推演更准确
```

### 平行人生推演流程

```
用户发起"如果当初..."
    │
    ▼
系统从记忆库提取情境
    │
    ▼
查询真实世界数据
    │
    ▼
构建推演上下文
    │
    ▼
生成推演节点 + 动态漫画
    │
    ▼
用户观看/互动
    │
    ▼
选择分支 → 继续推演
    │
    ▼
存入世界线树
```

### 遗憾处理流程

```
遗憾被识别
    │
    ▼
分类判断（可弥补/不可弥补）
    │
    ├── 可弥补 → 参考历史成功/失败路径 → 自然提及 → 用户决定
    │
    └── 不可弥补 → 关联场景触发 → 帮助释怀
    │
    ▼
融入世界线（遗憾视图筛选）
```

## 技术要点

### 数据安全

- 用户主密码 → PBKDF2派生 → 主密钥 + 设备密钥
- 所有敏感数据本地加密后上传
- 服务端只存密文，无法解密
- 微信机器人：7天授权周期，扫码获取临时密钥

### 记忆保留策略

| 层级 | 策略 |
|------|------|
| 关键片段 | 永久保留 |
| 原始对话 | 保留2年，之后压缩为摘要 |
| 时间线事件 | 永久保留 |
| 隐藏层 | 永久保留，用户不可见 |

### 多端同步

- 本地优先，所有数据先写本地
- 加密传输，云端只存密文
- 冲突解决：时间戳+操作类型
- 实时同步：画像更新、关键片段、时间线事件
